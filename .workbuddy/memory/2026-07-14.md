## Bug 排查：页面除"新增工单"外全空白 → **已修复**
- 症状：手机和 PC 都一样，菜单栏和底部 dock 正常，tab 切换正常但内容空白。
- 排查过程（大量 CDP 无头 Chrome 测试）：
  - ✅ 后端 API 正常（登录+带 token 调 dashboard/customers 均 200）
  - ✅ 排除了 JWT exp 秒/毫秒单位 bug（PyJWT 处理正确）
  - ✅ 排除了 Jinja2 渲染破坏
  - ✅ 移动端/桌面端无头测试均正常
  - ⚠️ 重启容器无效
- **根因：`tab-work`(新增工单 tab, L155) 缺一个 `</div>` 未闭合**，104 个 `<div>` 开但只有 103 个关。浏览器把所有后续 tab（tab-pending/salary/expenses/.../dashboard/calendar 共 18 个 tab）全部解析为 tab-work 的子节点。tab-work 本身 `display:none`，所有嵌套 tab 不可见。
- **修复：在 tab-work 末尾（L437 后）补一个 `</div>`**，关闭未闭合的 div。部署后用户手机/PC 均验证通过，所有 tab 正常显示。
- 教训：HTML div 未闭合是难以察觉的 bug——代码逻辑完全正常、DOM 内容存在但被父容器隐藏；Chrome 和手机浏览器容错处理不同所以无头测试通过但真实设备失败；逐层 CSS `computedStyle` 追踪到 `parent: display:none` 才定位。
