# worklog-system 全面切换 shadcn-vue + 移动端高度优化方案

> 目标：将现有前端从 Element Plus（CDN 模式）全面切换到 **Vite + Vue3 + TypeScript + Tailwind + shadcn-vue**，并以**手机优先**为设计核心，打造高质感、适合现场作业人员使用的工单系统。

本文档可直接提供给本地 AI 执行。

---

## 一、项目背景与约束

### 1.1 业务特点
- 系统名称：worklog-system（弱电/智能化/低压强电工程施工与维护业务管理系统）
- 主要用户：现场施工人员、技术员、项目经理
- **使用场景以手机为主，几乎不用电脑**
- 核心高频操作：新建工单、拍照上传、填写施工内容、查看待办、更新状态

### 1.2 技术约束
- 后端已切换为 Flask Blueprints，API 基本稳定
- 原前端为无构建 CDN + Element Plus，现需引入构建流程
- 最终部署到局域网 NAS，构建后输出静态文件即可
- 需要支持 PWA（可添加到主屏幕 + 基础离线能力）

### 1.3 目标
- 视觉达到高质感（干净、克制、现代）
- 手机体验优秀（大触控、清晰层级、操作路径短）
- 代码结构清晰，便于后续维护和扩展

## 四、详细实施步骤

### 阶段 1：初始化项目（必须严格按顺序）

1. 使用官方方式创建 Vite + Vue3 + TS 项目
2. 安装并配置 Tailwind CSS
3. 初始化 shadcn-vue（使用官方 CLI）
4. 配置 `components.json`，推荐使用 `new-york` 风格 + 石板色系（更高级）
5. 安装必要依赖：
   - `lucide-vue-next`
   - `clsx` + `tailwind-merge`（shadcn 需要）
   - `vite-plugin-pwa`
   - `pinia` + `vue-router`
6. 配置好路径别名 `@/`
7. 建立基础主题色（建议以深蓝/石板灰为主，避免花哨）

**主题建议（可调整）：**
- 主色：slate / zinc 系 + 一个品牌色（如 indigo 或 blue）
- 圆角：偏大一点（`rounded-xl` / `rounded-2xl`）
- 字体：系统字体栈优先，保证手机端清晰

### 阶段 2：搭建移动端基础骨架

必须完成以下组件：

#### 2.1 SafeArea 处理
```vue
<!-- 支持刘海屏和底部横条 -->
<meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover, maximum-scale=1">
```

使用 `env(safe-area-inset-top)` 和 `env(safe-area-inset-bottom)`。

#### 2.2 BottomNav.vue
- 固定底部
- 中间「新建」按钮视觉突出（可使用浮动圆形按钮）
- 当前路由高亮
- 适配安全区域

#### 2.3 MobileHeader.vue
- 左侧返回 / 标题
- 右侧操作按钮（筛选、更多等）
- 支持透明/固体背景切换

#### 2.4 AppShell / MobileLayout
- 统一处理顶部安全区 + 底部导航占位
- 页面内容区域可滚动
- 避免被底部导航遮挡

### 阶段 3：设计系统与基础组件微调

在 shadcn-vue 生成的组件基础上，针对手机做以下调整：

1. **Button**
   - 默认高度 ≥ 44px
   - 主要按钮使用较大字重和圆角
   - 增加 `size="xl"` 变体（方便现场点击）

2. **Input / Textarea / Select**
   - 高度加大
   - 字体不小于 16px（防止 iOS 自动缩放）
   - 错误状态清晰

3. **Card**
   - 作为手机端列表的主要载体（替代表格）
   - 增加按压态反馈

4. **Dialog → 优先使用 Drawer**
   - 手机端大量操作改用底部抽屉（Drawer）
   - 确认类弹窗可用 Dialog，但要适配小屏

5. **Badge / Status**
   - 工单状态颜色体系要统一且易区分（待处理、进行中、待验收、已完成、已取消等）

### 阶段 4：核心页面迁移优先级（严格按此顺序）

#### 优先级 1（必须先做好）
1. 登录页
2. 工作台（Dashboard）
3. 工单列表（卡片式）
4. **新建工单**（最重要，重点打磨）
5. 工单详情
6. 待办列表

#### 优先级 2
- 客户管理
- 项目管理
- 员工/人员
- 物料
- 个人中心

#### 优先级 3
- 财务相关
- 统计报表
- 系统设置

### 阶段 5：新建工单页面专项优化（核心体验）

新建工单是现场人员使用频率最高、也最容易感到麻烦的页面，必须重点优化：

**优化要求：**
- 采用分步表单（Stepper）或长表单 + 锚点，避免一屏信息过载
- 照片上传支持多图、压缩、预览、删除
- 常用字段提供历史记录 / 快捷选择
- 客户、项目、人员支持搜索选择（Combobox）
- 提交按钮固定在底部（或使用底部操作栏）
- 支持「保存草稿」（本地 + 后续可接离线）
- 必填项提示清晰，错误定位准确
- 大触控面积，减少误触

**推荐信息结构：**
1. 基本信息（客户、项目、地址、联系人）
2. 工作内容（类型、描述、人员）
3. 现场照片与附件
4. 其他（预计工时、备注等）

### 阶段 6：列表与详情移动端优化

**列表页：**
- 全部使用卡片列表，禁止在手机端使用传统表格
- 支持下拉刷新
- 支持常用筛选（状态、时间、负责人）—— 用底部抽屉实现筛选面板
- 卡片上直接展示关键信息 + 状态标签
- 左滑可考虑快捷操作（可选）

**详情页：**
- 顶部状态 + 关键操作按钮
- 内容分区清晰（基本信息、工作记录、照片、流转记录）
- 操作按钮优先放在底部固定栏

### 阶段 7：PWA 与离线基础能力

使用 `vite-plugin-pwa` 完成：

1. 正确的 manifest（名称、图标、主题色、display: standalone）
2. maskable 图标
3. 静态资源预缓存
4. 基础离线页面提示
5. 自动更新策略（registerType: 'autoUpdate'）
6. （可选进阶）新建工单支持断网暂存，恢复网络后同步

### 阶段 8：联调与替换

1. 封装统一的 API 请求层（基于现有后端接口）
2. 处理登录鉴权与路由守卫
3. 逐个页面替换旧前端
4. 保留旧版本一段时间做对比测试（可选）
5. 真机测试（iOS Safari + Android Chrome）重点验证：
   - 安全区域
   - 键盘弹起时的布局
   - 照片上传
   - 底部导航是否被遮挡
   - 横竖屏（以竖屏为主）

## 五、移动端设计规范（必须遵守）

1. **触控目标** ≥ 44px × 44px
2. **正文基础字号** ≥ 16px
3. **页面左右边距** 建议 16px 或 20px
4. **卡片间距** 保持呼吸感，避免拥挤
5. **主要操作** 尽量一屏内可达，或使用底部固定栏
6. **颜色** 保持克制，状态色语义清晰
7. **动画** 轻微、快速，避免过度装饰影响效率
8. **加载与空状态** 必须友好，不能白屏
