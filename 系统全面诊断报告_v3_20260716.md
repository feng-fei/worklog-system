### 1.3 核心业务数据流

```
【开单流程】
表单提交 → create_record → 生成工单号
                       → 解析临时工明细 → _sync_salary_records_for_work → SalaryRecord
                       → 解析设备/材料明细 → _sync_equipment_details
                                                    → RepairEquipment
                                                    → _adjust_material_stock → Material.stock扣减
                                                                              → MaterialStockLog
                       → 费用计算(人工+材料+设备+交通+其他+税) → total_fee
                       → payment_status=unpaid, paid_amount=0
                       → _log_operation
                       → 返回工单（含warnings库存告警）

【收款流程】
收款录入 → create_payment → PaymentRecord
                       → 更新 WorkRecord.paid_amount（累加）
                       → 更新 WorkRecord.payment_status（自动判断unpaid/partial/paid）
                       → _log_operation

【工资结算】
settle_salary → SalaryRecord.status=settled, paid_amount=payable_amount

【删除流程（v3修复后）】
delete_record → 删除照片+缩略图
              → 回滚设备物料库存 → 删除RepairEquipment
              → 删除SalaryRecord
              → PaymentRecord.record_id=NULL（解除关联）
              → PendingWork.related_record_id=NULL（解除关联）
              → 删除WorkRecord
              → _log_operation
```

### 1.4 权限模型现状

| 角色 | 可访问数据 | 可执行操作 |
|------|----------|-----------|
| **admin** | 全量数据：所有工单、财务、客户、物料、项目、统计 | 所有增删改操作 |
| **worker** | 自己相关的工单（staff_names包含自己/staff_name/created_by）；自己的待办、自己的工资；看不到支出和全公司财务统计 | 查看+创建工单（设计上应允许），不能删改财务数据 |
